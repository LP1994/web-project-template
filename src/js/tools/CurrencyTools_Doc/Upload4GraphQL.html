multipleFiles(operationName, query, variables, file4KeyName) → {FormData}

多文件上传
PS:
1、GraphQL上传请求(多文件)的规范
https://github.com/jaydenseric/graphql-multipart-request-spec#file-list
Parameters:
Name Type Description
operationName 字符串，默认值null(表示不传)，可选，这个字段的使用遵循GraphQL中的“operationName”的使用方法
query 字符串，字符串形式的GraphQL语句，必传
variables 对象，里头的数据格式是key-value形式的，其中涉及用于存放文件的那个key所对应的值必须是FormData的数据类型，
该FormData里头只允许接受三种数据类型的值：File、Blob、ArrayBuffer，必传。
如：
variables: {
filesA: formData,

PS：该工具在处理“formData”过程中的规范说明！请使用者注意看说明再使用！
1、在“formData”中的各个“key”都必须是唯一的！
3、在“formData”中，如果同一个“key”存放了多个文件，那也只取其第一个文件。
file4KeyName 字符串，用于说明在上面“variables”字段中用于存放文件的那个key的名字，必传
