write4Clip(data, events)

使用“Clipboard API”的write方法将一个内容写入系统剪切板
PS：
1、奇怪！！！浏览器竟然对写入系统剪切板的权限没有提示！！！不管其权限是何种状态！！！而且不能成功写入！！！
Parameters:
Name Type Description
data Blob、DataTransfer，一个写入系统剪切板的数据，必须
PS：
1、允许写入系统剪切板的数据类型如下：
text/plain、text/uri-list、text/csv、text/html、image/svg+xml、application/xml、text/xml、application/json
2、警告！为了安全起见，不受信任的脚本允许写入剪贴板的数据类型受到限制。
不受信任的脚本可以尝试利用本地软件中的安全漏洞，方法是将已知触发这些漏洞的数据放在剪贴板上。
events JSON对象，里头都是各个事件，可选
{
success: 函数，将一个内容成功写入系统剪切板时执行，可选

fail: 函数，将一个内容写入系统剪切板失败时执行，可选

denied: 函数，当权限状态为“拒绝授权”时执行，可选

stateChange: 函数，当权限状态改变时执行，会有一个参数(permissionStatus对象)，可选

error: 函数，报错时执行的函数，会有一个参数(error)，可选
